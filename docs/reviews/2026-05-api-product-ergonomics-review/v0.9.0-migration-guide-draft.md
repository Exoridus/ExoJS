# ExoJS v0.8.x → v0.9.0 Migration Guide

**Status:** Draft — internal review document  
**Authority:** Source at HEAD (`main`, v0.8.4 → v0.9.0 changes implemented)  
**Public destination:** `site/src/content/guide/migration/v0.9.0.mdx`

---

## Overview

v0.9.0 is the API consolidation milestone for ExoJS. The public surface has been restructured,
deduplicated, and documented as part of a focused seven-pass API/product review. Most changes are
mechanical renames or moves — the engine behavior and scene/rendering model are unchanged.

**Estimated migration time:** 15–30 minutes for a typical project, mostly search-and-replace.

### Breaking change summary

| Area | Type | Impact |
|---|---|---|
| `ApplicationOptions` — grouped structure | **Breaking** | All projects |
| `Loader` option renames (`basePath`, `fetchOptions`) | **Breaking** | All projects using assets |
| `Application.sceneManager` → `Application.scene` | **Breaking** | All projects |
| `SceneManager.scene` → `SceneManager.currentScene` | **Breaking** | Any code reading current scene |
| `Scene.getParticipationPolicy()` removed | **Breaking** | Rare — internal use only |
| `SceneNode.parentNode` removed | **Breaking** | Any code reading parent |
| `SceneNode.bounds` / `.globalTransform` / `.localBounds` removed | **Breaking** | Any code using these aliases |
| `SceneNode.setCullable()` removed | **Breaking** | Rare |
| `RenderNode.setCacheAsBitmap()` removed | **Breaking** | Rare |
| `RenderNode.setFilters()` removed | **Breaking** | Rare |
| `Text.setText()` / `Text.setStyle()` removed | **Breaking** | Common |
| `Color.red` / `.green` / `.blue` / `.alpha` removed | **Breaking** | Rare |
| `Tween.to()` — `NumericKeys<T>` constraint | **Breaking (TypeScript)** | TypeScript callers only |

### Additive / behavior-only changes

- `skewX`, `skewY`, `setSkew(x, y)` added to all `SceneNode` subclasses
- `TweenManager.sequence()` added
- `Scene.tweens` proxy auto-disposes on scene destroy
- `pauseOnHidden = true` no longer accumulates a delta spike on resume
- `backend.stats.rawFrameDeltaMs` available for frame profiling
- `Text.ready: Promise<void>` for font load awaiting
- `View.screenToWorld(screenX, screenY, canvasWidth, canvasHeight): PointLike` / `worldToScreen(worldX, worldY, canvasWidth, canvasHeight): PointLike` helpers added
- Typed asset system: `Asset<T>`, `Assets<M>`, `LoadingQueue`

## 1. ApplicationOptions — grouped structure

This is the most impactful breaking change. All configuration is now organized into named groups.

### What moved where

| Old (flat top-level) | New location | Notes |
|---|---|---|
| `canvas` (HTMLCanvasElement) | `canvas.element` | Key renamed |
| `width` | `canvas.width` | Moved into group |
| `height` | `canvas.height` | Moved into group |
| `resourcePath` | `loader.basePath` | Moved **and renamed** |
| `requestOptions` | `loader.fetchOptions` | Moved **and renamed** |
| `cache` | `loader.cache` | Moved into group |
| `gamepadDefinitions` | `input.gamepadDefinitions` | Moved into group |
| `gamepadSlotStrategy` | `input.gamepadSlotStrategy` | Moved into group |
| `pointerDistanceThreshold` | `input.pointerDistanceThreshold` | Moved into group |
| `debug` | `rendering.debug` | Moved into group |
| `webglAttributes` | `rendering.webglAttributes` | Moved into group |
| `spriteRendererBatchSize` | `rendering.spriteRendererBatchSize` | Moved into group |
| `particleRendererBatchSize` | `rendering.particleRendererBatchSize` | Moved into group |

| Still top-level | Notes |
|---|---|
| `clearColor` | Unchanged |
| `backend` | Unchanged |

### New options in v0.9.0

| Option | Type | Default | Notes |
|---|---|---|---|
| `canvas.pixelRatio` | `number` | `1` | HiDPI backing buffer scaling |
| `canvas.tabIndex` | `number` | `-1` | Was hardcoded |
| `canvas.imageRendering` | `'auto' \| 'pixelated' \| 'crisp-edges'` | `'auto'` | CSS hint for pixel-art |
| `loader.cacheStrategy` | `CacheStrategy` | `CacheFirstStrategy` | Was unreachable before |
| `loader.concurrency` | `number` | `6` | Was unreachable before |

> **Note on `canvas.tabIndex`:** If you pass an existing canvas element and omit `canvas.tabIndex`, ExoJS preserves an existing `tabindex` attribute rather than overwriting it; otherwise it defaults to `-1`.

### Before / after

**Minimal app — no change:**
```ts
// BEFORE and AFTER — unchanged
const app = new Application();
```

**Typical game app:**
```ts
// BEFORE
const app = new Application({
  canvas,
  width: 1280,
  height: 720,
  clearColor: Color.black,
  resourcePath: 'assets/',
  cache: new IndexedDbStore('mygame'),
  gamepadDefinitions: [myPad],
  gamepadSlotStrategy: 'compact',
  spriteRendererBatchSize: 8192,
  webglAttributes: { antialias: true },
  debug: true,
});

// AFTER
const app = new Application({
  clearColor: Color.black,

  canvas: {
    element: canvas,
    width: 1280,
    height: 720,
  },

  loader: {
    basePath: 'assets/',
    cache: new IndexedDbStore('mygame'),
  },

  input: {
    gamepadDefinitions: [myPad],
    gamepadSlotStrategy: 'compact',
  },

  rendering: {
    spriteRendererBatchSize: 8192,
    webglAttributes: { antialias: true },
    debug: true,
  },
});
```

**Loader options — rename:**
```ts
// BEFORE
const app = new Application({
  resourcePath: 'assets/',
  requestOptions: { mode: 'same-origin' },
  cache: new IndexedDbStore('mygame'),
});
app.loader.setConcurrency(4); // concurrency was unreachable at construction time

// AFTER
const app = new Application({
  loader: {
    basePath: 'assets/',
    fetchOptions: { mode: 'same-origin' },
    cache: new IndexedDbStore('mygame'),
    cacheStrategy: new NetworkOnlyStrategy(),
    concurrency: 4,
  },
});
```

**Pixel-art game with HiDPI canvas:**
```ts
// BEFORE — entirely user responsibility
const app = new Application({ canvas, width: 320, height: 240 });
canvas.style.imageRendering = 'pixelated';
canvas.width = Math.round(320 * window.devicePixelRatio);
canvas.height = Math.round(240 * window.devicePixelRatio);
canvas.style.width = '320px';
canvas.style.height = '240px';

// AFTER
const app = new Application({
  canvas: {
    element: canvas,
    width: 320,
    height: 240,
    pixelRatio: window.devicePixelRatio,
    imageRendering: 'pixelated',
  },
});
```

**WebGPU backend — cleaner top level:**
```ts
// BEFORE — WebGPU user saw debug/webglAttributes/batch sizes at top level (silently ignored)
const app = new Application({
  width: 1920,
  height: 1080,
  backend: { type: 'webgpu' },
});

// AFTER — no change needed; top level is clean for WebGPU users
const app = new Application({
  canvas: { width: 1920, height: 1080 },
  backend: { type: 'webgpu' },
});
```

> **Note:** There are no compatibility shims. The old flat-top-level shape is fully removed.
> TypeScript will error on any unrecognized top-level key.

## 2. Application subsystem rename

`app.sceneManager` is renamed to `app.scene`, matching the naming pattern used by `app.input`
and `app.audio`.

Simultaneously, `SceneManager.scene` (the getter returning the active scene) is renamed to
`SceneManager.currentScene` to avoid the awkward `app.scene.scene` compound.

```ts
// BEFORE
app.sceneManager.setScene(new TitleScene());
app.sceneManager.pushScene(new PauseScene());
app.sceneManager.popScene();

const active: Scene | null = app.sceneManager.scene;
const stack: Scene[] = app.sceneManager.scenes;

// AFTER
app.scene.setScene(new TitleScene());
app.scene.pushScene(new PauseScene());
app.scene.popScene();

const active: Scene | null = app.scene.currentScene;
const stack: Scene[] = app.scene.scenes;
```

`app.input` and `app.audio` are already the canonical short names from a prior pass —
no change required for those.

## 3. Scene API cleanup

### `getParticipationPolicy()` removed

The method returned a one-field wrapper `{ mode: this.stackMode }`. It has been removed.
Read `scene.stackMode` directly.

```ts
// BEFORE
const policy = scene.getParticipationPolicy();
if (policy.mode === 'overlay') { ... }

// AFTER
if (scene.stackMode === 'overlay') { ... }
```

`setParticipationPolicy()` is **not removed** — it remains as the fluent API for batch-setting
scene participation, particularly useful with `PushSceneOptions`.

### Scene stack modes

`SceneStackMode` values and semantics are unchanged: `'overlay'`, `'modal'`, `'opaque'`.

## 4. Public object API cleanup

### SceneNode — removed property aliases

```ts
// BEFORE                              → AFTER
node.parentNode                        → node.parent
node.bounds                            → node.getBounds()
node.globalTransform                   → node.getGlobalTransform()
node.localBounds                       → node.getLocalBounds()
node.setCullable(value)                → node.cullable = value
```

**Behavior note:** `getBounds()`, `getGlobalTransform()`, and `getLocalBounds()` are
lazy-computed methods (cache rebuilds). The removed property aliases were thin delegates
to the same methods. The semantic result is identical; only the call site changes.

### RenderNode — removed method duplicates

```ts
// BEFORE                              → AFTER
node.setCacheAsBitmap(value)           → node.cacheAsBitmap = value
node.setFilters(filters)               → node.filters = filters
```

`node.filters` is a setter that replaces the entire filter array. The filter management
API includes:

```ts
node.filters = [new BlurFilter(), new ColorFilter()]; // replace all
node.addFilter(filter);                               // append
node.removeFilter(filter);                            // remove one
node.clearFilters();                                  // remove all
```

### Text — removed setter methods

```ts
// BEFORE                              → AFTER
text.setText('Hello')                  → text.text = 'Hello'
text.setStyle({ fontSize: 24 })        → text.style = { fontSize: 24 }
```

### Color — removed long-name aliases

```ts
// BEFORE                              → AFTER
color.red                              → color.r
color.green                            → color.g
color.blue                             → color.b
color.alpha                            → color.a
```

`Color.r`, `.g`, `.b`, `.a` are the only property names. The `.red`, `.green`, `.blue`, `.alpha`
names have been removed.

### Policy overview

The resolved naming policy for public objects in ExoJS:

| Pattern | Shape | Example |
|---|---|---|
| Stored mutable state | getter/setter property | `node.visible`, `node.x`, `node.cullable` |
| Lazy-computed result | `get*()` method only | `node.getBounds()`, `node.getGlobalTransform()` |
| Multi-arg or chained mutation | `set*(...)` returning `this` | `node.setPosition(x, y)`, `node.setSkew(x, y)` |
| Lifecycle | `void` | `destroy()`, `stop()` |

## 5. Loader / Assets

v0.9.0 ships a typed asset system. The low-level `loader.load(url, Type)` API still works, but
the new typed references are the recommended path for any project with more than a handful of assets.

### Typed asset references — `Asset<T>` and `Assets<M>`

```ts
import { Asset, Assets } from '@codexo/exojs';

// Single typed reference — config-object form
const heroTexture = new Asset({ type: 'texture', source: 'images/hero.png' });

// Typed manifest (key → resource type)
// Default built-in types: texture, sound, music, json
const titleAssets = new Assets({
  logo:   { type: 'texture', source: 'images/logo.png' },
  music:  { type: 'music',   source: 'audio/title.ogg' },
  levels: { type: 'json',    source: 'data/levels.json' },
});
```

### Loading a single asset

`loader.load()` returns the loaded resource directly when given a single `Asset<T>`:

```ts
const tex: Texture = await loader.load(heroTexture);
```

### Loading an asset container

When given an `Assets<M>` manifest, `loader.load()` returns a `LoadingQueue`. Awaiting the
queue resolves to a typed results map — `Assets` itself holds only config, not loaded resources:

```ts
const queue: LoadingQueue = loader.load(titleAssets);

queue.onProgress.add(({ loaded, total }) => {
  progressBar.width = (loaded / total) * 300;
});

const results = await queue;

// Access typed resources from the resolved result
const logo: Texture = results.logo;
const music: Music   = results.music;
```

### Loader option renames

These renames apply both to `LoaderOptions` (passed to `new Loader(...)`) and to
`ApplicationOptions.loader`:

| Old name | New name |
|---|---|
| `resourcePath` | `basePath` |
| `requestOptions` | `fetchOptions` |

```ts
// BEFORE
const loader = new Loader({ resourcePath: '/assets/', requestOptions: { mode: 'same-origin' } });

// AFTER
const loader = new Loader({ basePath: '/assets/', fetchOptions: { mode: 'same-origin' } });
```

Post-construction access:
```ts
app.loader.basePath = '/assets/v2/';
app.loader.fetchOptions = { mode: 'cors' };
```

### Cache strategies

Built-in strategies: `CacheFirstStrategy` (default — check IDB before network) and `NetworkOnlyStrategy` (bypass cache entirely).

```ts
const app = new Application({
  loader: {
    basePath: 'assets/',
    cache: new IndexedDbStore('mygame'),
    cacheStrategy: new CacheFirstStrategy(),  // default — check IDB before network
    // or: new NetworkOnlyStrategy()           — bypass cache entirely
  },
});
```

### Custom asset types

```ts
// Register a custom type handler
loader.register(MyShader, new MyShaderFactory());

// Load it like any built-in type (requires AssetDefinitions augmentation for full type safety)
const ref = new Asset({ type: 'shader', source: 'shaders/custom.glsl' });
await loader.load(ref);
```

### Old low-level API — still supported

`loader.load(Texture, 'path/to/image.png')` continues to work. The low-level API is not removed.
The typed `Asset`/`Assets` system is additive.

## 6. Tween changes

### Managed tweens restart correctly

Before v0.9.0, calling `tween.start()` after a tween had completed or been stopped would reset
internal state to Active but the `TweenManager` would no longer drive the tween — because
`complete()` and `stop()` evict the tween from the manager. The tween appeared to restart but
produced no animation.

This is fixed: `start()` now re-registers the tween with its manager when called after eviction.

```ts
// Ping-pong pattern — now works correctly across multiple rounds
const t = app.tweens.create(sprite.position)
  .to({ x: 400 }, 1)
  .onComplete(() => {
    t.to({ x: 0 }, 1).start(); // re-registers with manager — correct in v0.9.0
  })
  .start();
```

### `NumericKeys<T>` constraint on `to()`

`Tween.to()` now accepts only keys whose type is `number` on the target object. Passing a
non-numeric key is a TypeScript compile error.

```ts
const pos = { x: 0, y: 0, name: 'origin' };
const tween = app.tweens.create(pos);

tween.to({ x: 100, y: 200 }, 1);     // ✓ — both numeric
tween.to({ name: 'target' }, 1);      // ✗ — TypeScript error: 'name' is not numeric
```

If you have targets with mixed types and need to tween a subset of numeric keys, this was
already the correct runtime behavior (non-numeric properties were skipped with a console
warning). The change makes the restriction visible at compile time.

### `TweenManager.sequence()` added

Creates a sequenced array of tweens linked via `chain()`. Returns the first tween.

```ts
// BEFORE — manual chaining
const a = app.tweens.create(node).to({ x: 100 }, 0.5);
const b = app.tweens.create(node).to({ y: 200 }, 0.5);
a.chain(b);
a.start();

// AFTER — sequence helper
app.tweens.sequence([
  app.tweens.create(node).to({ x: 100 }, 0.5),
  app.tweens.create(node).to({ y: 200 }, 0.5),
]).start();
```

### Scene-scoped tweens — auto-dispose

`scene.tweens` is a `SceneTweens` proxy that automatically stops all tracked tweens when the
scene is destroyed. Tweens created via `scene.tweens.create()` or added via `scene.tweens.add()`
are tracked automatically.

```ts
class MyScene extends Scene {
  override async load(): Promise<void> {
    // Tween is stopped automatically when MyScene is destroyed
    this.tweens.create(this.hero.position)
      .to({ x: 500 }, 2)
      .repeat(Infinity)
      .yoyo()
      .start();
  }
}
```

### `@internal` methods filtered from docs

`Tween._attachManager()` no longer appears in generated API documentation.
The public API surface of `Tween` is unchanged.

### Timing — seconds are canonical

No change. All tween durations are in seconds. This is consistent with the particle system.

## 7. Application loop / timing behavior

These are **behavior fixes and additive stats** — no public API change required for most projects.

### `pauseOnHidden` delta spike — fixed

When `pauseOnHidden = true` (default: `false`) and the tab returns from hidden, the previous
version accumulated the full hidden duration as a single massive frame delta. This caused
simulations to jump forward by seconds on resume.

v0.9.0 restarts the frame clock in the early-return path, discarding the hidden interval:

```ts
app.pauseOnHidden = true;
// Tab is hidden for 30 seconds
// Tab becomes visible again
// ✓ Next frame delta is ~16ms, not ~30,000ms
```

### Internal delta clamp (`maxDeltaMs`)

An internal cap of 100 ms is applied to the frame delta before it reaches scene `update()`.
This prevents simulation explosions when a single frame takes longer than expected
(garbage collection pause, first-frame startup). Tween, particle, and scene update code
all receive a clamped delta.

This is not a public option in v0.9.0. No configuration is needed.

### `backend.stats.rawFrameDeltaMs` — new stat

The unclamped frame delta (before the 100 ms cap) is available for profiling:

```ts
app.onFrame.add(() => {
  console.log('raw delta:', app.backend.stats.rawFrameDeltaMs);
  console.log('clamped delta fed to update:', /* visible via Time.milliseconds in scene.update */);
});
```

### `fpsLimit` / `backgroundFpsLimit` — not in v0.9.0

These were proposed during the review but explicitly deferred. `pauseOnHidden = true` is the
correct primitive for background-tab scenarios. A render-only `fpsLimit` is deferred to a
future pass when concrete user demand exists.

## 8. Transform / Skew — new API

Skew support is new in v0.9.0. All `SceneNode` subclasses (sprites, containers, text, etc.)
now support independent horizontal and vertical shear.

```ts
// Skew properties (degrees)
node.skewX = 15;     // horizontal shear — positive leans top edge right
node.skewY = -10;    // vertical shear — negative leans left edge up

// Compound setter — returns this for chaining
node.setSkew(15, -10);
node.setSkew(15);    // sets both axes to 15°

// Reset
node.setSkew(0);
```

### Behavior notes

- Units are **degrees**, consistent with `node.rotation`.
- Non-zero skew means the node is no longer axis-aligned. `node.isAlignedBox` returns `false`
  when either `skewX !== 0` or `skewY !== 0`, which gates AABB fast-paths in hit-testing and
  bounds computation.
- `Sprite.contains()` works correctly for skewed sprites — a bug where skewed sprites at
  axis-aligned rotations used the wrong fast-path has been fixed.
- Skew composes correctly with rotation and scale in the transform matrix (fused RS+Skew
  computation via `Math.tan` shear).
- Bounds and rendering are both correct under skew.

### Example — sign board with lean

```ts
const sign = new Sprite(loader.get(Texture, 'sign.png'));
sign.skewX = 12;  // lean the sign slightly to the right
scene.root.addChild(sign);
```

## 9. Collision — documentation and correctness updates

No new public API surface was added. These changes fix documentation and clarify semantics.

### `projectionV` is the minimum-translation vector

`CollisionResponse.projectionV` is the vector you apply to separate shapes with minimum
displacement. A previously misleading JSDoc comment (claiming it was a zero vector) has been
corrected.

```ts
const response = rect.collidesWith(player);
if (response) {
  // Push player out of rect with minimum displacement
  player.move(response.projectionV.x, response.projectionV.y);
}
```

### `projectionN` is the collision normal

`CollisionResponse.projectionN` is the **unit** normal of the minimum-translation axis.
Use it for directional classification (which side was hit?):

```ts
const response = wall.collidesWith(ball);
if (response) {
  const { projectionN, projectionV } = response;

  // Separate
  ball.move(projectionV.x, projectionV.y);

  // Reflect velocity
  const dot = ball.velocity.x * projectionN.x + ball.velocity.y * projectionN.y;
  ball.velocity.x -= 2 * dot * projectionN.x;
  ball.velocity.y -= 2 * dot * projectionN.y;
}
```

### `collidesWith()` null return — two cases

`collidesWith()` returns `null` in two distinct situations. The JSDoc now names both:

1. The shapes **do not overlap**.
2. The **shape-pair combination does not support** full response generation
   (e.g. `Line` vs. any shape; `Ellipse` vs. `Ellipse` or `Polygon`).

Use `intersectsWith()` for a fast boolean-only test that works for all pairs.

### Swept collision helpers

These functions were already implemented and exported. They are now documented in the guide.

```ts
import {
  sweepRectangle,
  sweepCircleVsRectangle,
  sweepCircleVsCircle,
  sweepRectangleAgainst,
  sweepCircleAgainst,
  substepSweep,
} from '@codexo/exojs';
```

`sweepCircleVsRectangle` uses full Minkowski rounded-rectangle formulation (all four flat
faces + four corner arcs tested independently — no false corner collisions).

`substepSweep` is a generator — consume with `for...of`:

```ts
for (const step of substepSweep(x, y, dx, dy, 8)) {
  // test each sub-step position
}
```

See the Collision guide for movement and anti-tunneling recipes.

## 10. API docs — `@internal` filtering and Vector properties

### `@internal` filtering

Methods marked `/** @internal */` in source no longer appear in generated API pages.
Affected in this release: `Tween._attachManager()`.

This is a documentation-only change. The methods still exist in source (they are used
by the engine internally); they are simply not surfaced in the public API reference.

### Vector — inherited properties now visible

The generated `vector.mdx` API page now includes inherited `AbstractVector` properties
(`angle`, `length`, `lengthSq`) alongside `x` and `y`.

`AbstractVector.angle` (get/set) is the only way to read or set vector direction in
degree-free radians. It is used internally by `PolarVector`. It is retained as-is.

`AbstractVector.magnitude` does not exist and was never added. Use `length` and `lengthSq`.
