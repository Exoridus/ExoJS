# ExoJS v0.9.0 Release Plan

**Status:** Draft ‚Äî internal planning document  
**Date:** 2026-05-21  
**Current version:** `0.8.4` (package `@codexo/exojs`)  
**Target version:** `0.9.0`

---

## 1. Release scope summary

v0.9.0 is the **API consolidation milestone** for ExoJS. Seven focused API/product review
passes ‚Äî covering application construction, asset loading, scene/object APIs, tween system,
loop timing, transform completeness, and collision documentation ‚Äî have been completed,
implemented, and verified against the test suite (1 423 tests, 105 suites, all passing at HEAD).

This release makes breaking changes to the public API surface that would be costly to defer past
1.0. All changes are clean breaks with no compatibility shims, per the project's pre-1.0 policy.
After this release the API surface is stable enough to begin the 1.0.0 stabilization track.

## 2. Breaking changes

### 2.1 `ApplicationOptions` ‚Äî grouped structure

**Impact: all projects**

The flat 14-option `ApplicationOptions` interface is replaced by a grouped structure.

| Group | Contains |
|---|---|
| Top-level | `clearColor`, `backend` |
| `canvas` | `element`, `width`, `height`, `pixelRatio`, `tabIndex`, `imageRendering` |
| `loader` | `basePath`, `fetchOptions`, `cache`, `cacheStrategy`, `concurrency` |
| `rendering` | `debug`, `webglAttributes`, `spriteRendererBatchSize`, `particleRendererBatchSize` |
| `input` | `gamepadDefinitions`, `gamepadSlotStrategy`, `pointerDistanceThreshold` |

New exported interfaces: `CanvasApplicationOptions`, `RenderingApplicationOptions`,
`InputApplicationOptions`.

The `LoaderOptions` interface (from `Loader.ts`) is re-exported and used directly as the
`loader` group type.

### 2.2 `LoaderOptions` ‚Äî field renames

**Impact: all projects using the asset system**

| Old | New |
|---|---|
| `resourcePath` | `basePath` |
| `requestOptions` | `fetchOptions` |

These renames apply to both `new Loader(options)` and `ApplicationOptions.loader`.

### 2.3 `Application.sceneManager` ‚Üí `Application.scene`

**Impact: all projects**

The `sceneManager` property is renamed to `scene`, consistent with the naming pattern of
`app.input` and `app.audio`.

### 2.4 `SceneManager.scene` ‚Üí `SceneManager.currentScene`

**Impact: any code reading the active scene**

The getter returning the active top-of-stack scene is renamed to `currentScene` to avoid
the `app.scene.scene` compound after the Application rename.
`SceneManager.scenes` (the full stack array) is unchanged.

### 2.5 `Scene.getParticipationPolicy()` removed

**Impact: near-zero ‚Äî no examples, no tests referenced it**

Replaced by direct `scene.stackMode` read. `setParticipationPolicy()` is retained.

### 2.6 `SceneNode` ‚Äî removed property aliases

**Impact: code reading bounds or parent**

| Removed | Replacement |
|---|---|
| `node.parentNode` | `node.parent` |
| `node.bounds` | `node.getBounds()` |
| `node.globalTransform` | `node.getGlobalTransform()` |
| `node.localBounds` | `node.getLocalBounds()` |
| `node.setCullable(v)` | `node.cullable = v` |

### 2.7 `RenderNode` ‚Äî removed method duplicates

| Removed | Replacement |
|---|---|
| `node.setCacheAsBitmap(v)` | `node.cacheAsBitmap = v` |
| `node.setFilters(f)` | `node.filters = f` |

### 2.8 `Text` ‚Äî removed setter methods

| Removed | Replacement |
|---|---|
| `text.setText(value)` | `text.text = value` |
| `text.setStyle(style)` | `text.style = style` |

### 2.9 `Color` ‚Äî long-name aliases removed

| Removed | Replacement |
|---|---|
| `color.red` | `color.r` |
| `color.green` | `color.g` |
| `color.blue` | `color.b` |
| `color.alpha` | `color.a` |

### 2.10 `Tween.to()` ‚Äî `NumericKeys<T>` constraint

**Impact: TypeScript callers passing non-numeric keys**

`Tween.to()` now enforces at compile time that all keys in the target map correspond to
numeric properties on the target type. Runtime behavior is unchanged (non-numeric keys were
already skipped with a warning); this change makes the restriction visible at compile time.

## 3. New capabilities

### 3.1 Typed asset system

- `Asset<T>` ‚Äî typed reference to a single loadable resource
- `Assets<M>` ‚Äî typed manifest mapping string keys to asset definitions
- `LoadingQueue` ‚Äî returned by `loader.load()`, supports `onProgress` signal and
  `await`-able
- `AssetDefinitions` ‚Äî type for defining asset-type-to-source maps
- `CacheStore`, `CacheStrategy` ‚Äî pluggable persistence (IDB-backed via `IndexedDbStore`)
- `CacheFirstStrategy`, `NetworkOnlyStrategy` built-in policies
- `loader.register(Type, factory)` ‚Äî custom asset type handlers
- `getIdentityKey` ‚Äî config-sensitive cache identity for reproducible cache lookups
- `loader.unload(asset)` ‚Äî respects `getIdentityKey` for correct invalidation
- `loader.concurrency` ‚Äî background queue parallelism (default: 6)

### 3.2 Canvas and application setup

- `canvas.pixelRatio` ‚Äî HiDPI backing buffer scaling, engine-managed
- `canvas.tabIndex` ‚Äî configurable (was hardcoded to -1)
- `canvas.imageRendering` ‚Äî CSS rendering hint for pixel-art games
- `loader.cacheStrategy` and `loader.concurrency` ‚Äî now reachable at construction time
  (were previously invisible through `ApplicationOptions`)
- `Text.ready: Promise<void>` ‚Äî font load awaiting
- `View.screenToWorld()` / `worldToScreen()` helpers

### 3.3 Skew transforms

- `SceneNode.skewX` / `SceneNode.skewY` ‚Äî getter/setter, degrees
- `SceneNode.setSkew(x, y)` ‚Äî compound setter, returns `this`
- `SceneNode.isAlignedBox` ‚Äî correctly accounts for skew
- `Sprite.contains()` bug fixed (used wrong fast-path gate for skewed sprites)
- Correct transform composition (fused RS+Skew via `Math.tan` shear)
- Skew dirty-flag (`SceneNodeTransformFlags.Skew`) wired into transform and bounds cascade

### 3.4 Collision improvements

- `sweepCircleVsRectangle` V2 ‚Äî full Minkowski rounded-rectangle with corner arcs
  (replaces the simpler expanded-AABB approach)
- Stale JSDoc on `getCollisionRectangleRectangle` corrected (previously claimed `projectionN`
  / `projectionV` were zero vectors; both are computed correctly)
- `collidesWith()` JSDoc names both null cases (no overlap; unsupported pair)
- Collision guide expanded with separation recipes, directional response, swept collision
  helpers, and anti-tunneling patterns

### 3.5 Tween improvements

- Managed tweens re-register with `TweenManager` on `start()` after eviction ‚Äî ping-pong
  patterns now work correctly across multiple rounds
- Dead `_startFired = false` line removed from repeat handling
- `TweenManager.sequence([...])` ‚Äî chains an array of tweens via `chain()`, returns first
- `SceneTweens` auto-dispose proxy ‚Äî tweens created via `scene.tweens` are stopped
  automatically when the scene is destroyed
- `Tween._attachManager()` no longer surfaced in public API docs

### 3.6 Loop stability

- `pauseOnHidden = true` no longer accumulates the full hidden-tab interval as a delta spike
  on resume (`_frameClock.restart()` called in the early-return path)
- Internal `maxDeltaMs = 100` clamp applied before scene update ‚Äî prevents simulation
  explosions from GC pauses or first-frame startup
- `backend.stats.rawFrameDeltaMs` ‚Äî unclamped delta exposed for profiling
- Async `update()` / `draw()` dev-mode warning (when scene methods return a Promise)
- Scene `root` blank-screen trap ‚Äî dev-mode warning added when root is empty

### 3.7 Signal performance

- Signals refactored to direct function pointers with dispatch-guard, removing per-dispatch
  context allocations (commit `88156a2`)

## 4. Deferred topics

The following are explicitly **not** part of v0.9.0. They are recorded here to prevent scope
creep and to document the reasons.

### Rendering

| Topic | Status | Notes |
|---|---|---|
| `fpsLimit` | Deferred | Defer until concrete user demand; belongs in `rendering` or `loop` group |
| `backgroundFpsLimit` | **Rejected concept** | Browser throttling defeats it; `pauseOnHidden` is the right primitive |
| Fixed-timestep loop | Deferred | Requires a concrete physics integration requirement; `maxDeltaMs` is sufficient for now |
| BlendMode Overlay / HardLight / Difference | Deferred | Low demonstrated need; post-0.9.0 creative rendering pass |
| RenderInstructions / batching architecture | Deferred | Internal engine optimization |
| Profiling overlay / DevTools integration | Deferred | `rawFrameDeltaMs` lays the stat foundation; UI tooling is a separate investment |

### Gameplay / Runtime

| Topic | Status | Notes |
|---|---|---|
| `CollisionWorld` / collision registry | Deferred | Must be co-designed with physics; `Quadtree<T>` is the user-accessible spatial primitive |
| `onCollisionEnter/Stay/Exit` events | Deferred | Requires `CollisionWorld` |
| Fixed-step physics (Box2D style) | Deferred | Physics integration will add a fixed-step sub-loop inside the RAF tick post-0.9.0 |
| Tilemap / TileMap rendering | Deferred (Extras) | `@codexo/exojs/extras/tileset` track post-0.9.0 |
| 9-slice sprite primitive | Deferred (Extras) | `@codexo/exojs/extras/9slice` track |
| Scene stacking preset constants | Optional | `SceneStackMode` string union + docs is sufficient; named constants are convenience |

### Platform APIs

| Topic | Status | Notes |
|---|---|---|
| WebHID / extended gamepad extras | Deferred | Custom mappings already configurable |
| Microphone / live audio capture helpers | Deferred (Extras) | `MediaStream` duck-typing suffices for now |
| Webcam / getUserMedia | Deferred (Extras) | Out of core scope |
| WASM audio (BeatDetector Stage 2/3) | Deferred | Post-0.9.0 audio optimization |
| Networking / WebSocket / WebRTC | **Rejected for core** | Separate package `@codexo/exojs-networking` after 1.0 |

### Package Architecture

| Topic | Status | Notes |
|---|---|---|
| Extras subpath governance | Deferred | Must define peer-dependency range, support ownership, quality bar first |
| `@codexo/exojs/extras/collision` | Deferred | `CollisionWorld` co-designed with physics |
| `@codexo/exojs-pathfinding` | Deferred | Separate package |
| Playground / site Monaco integration | 1.0 requirement | Committed in project docs plan |
| Export tiering / `@stable/@advanced/@internal` CI enforcement | Deferred | Documentation governance task |

## 8. Post-release priorities

After v0.9.0 is tagged, the recommended immediate next focus areas are:

1. **Extras governance decision** ‚Äî define the quality bar, peer-dep range, and ownership model
   for `@codexo/exojs/extras/*` before any Extras package ships.
2. **Physics integration planning** ‚Äî co-design `CollisionWorld` with the fixed-step sub-loop
   that physics will require; this unblocks `onCollisionEnter/Stay/Exit`.
3. **Playground / Monaco site integration** ‚Äî committed as a 1.0 requirement in the docs plan.
4. **BlendMode creative rendering pass** ‚Äî Overlay, HardLight, Difference are low-effort
   additions once v0.9.0 is out.
5. **`fpsLimit` mutable property** ‚Äî add when concrete user demand exists; `rendering` group
   is the right namespace.
