# v0.17 Slice D (remainder) — Pause-Input Policy, Modal Capture, Transition Gating — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Finish v0.17 Slice D (the pause-API _core_ — `SceneState.Paused`, `SceneDirector.pause()`/`resume()`, signals — already shipped in PR #392). This plan adds the three remaining pieces: a `when: 'active'|'paused'|'always'` availability policy on `SceneInputs` bindings with press/release edge rules, `SceneInteraction.capture(root)` (modal pointer confinement), and director-owned transition input gating (`_inputGateDepth`).

**Architecture:** `SceneInputs`/`SceneInteraction` currently hold only an `Application` reference — they gain a `() => SceneState` reader (supplied by `SceneScope`, which already owns `_state`) so they can gate dispatch without a circular class dependency. `SceneDirector` gains an internal `_inputGateDepth` counter, open only while an explicit fade transition (`_transition !== null`) is in flight, plus an `@internal` getter both facades and `InteractionManager` consult. `SceneInputs` wraps each `when`-decorated binding's four callbacks (`onStart`/`onActive`/`onStop`/`onTrigger`) with a policy + edge-tracking check. `SceneInteraction.capture()` is a thin, order-preserving wrapper over the interaction manager's _already-existing_ `pushInputCapture`/`popInputCapture` stack (confirmed present and complete — no manager-level capture-stack work needed). Dispatch gating for interaction (`Active`/`Paused` only, plus the same transition gate) is enforced once, at the top of `InteractionManager.update()` — the single per-frame pointer-dispatch chokepoint.

**Tech Stack:** TypeScript (strict), Vitest. Builds on `SceneDirector`/`SceneScope`/`SceneInputs`/`SceneInteraction`/`InteractionManager` from Slices A–C (merged, PR #396 / commit `23136582` is the base for this branch).

## Global Constraints

- Clean breaks only — no deprecated aliases, no shims (pre-1.0 policy).
- `app.input` bindings (non-scene) are **never** affected by `when` policy or transition gating — only `this.inputs`/`this.interaction` (scene-bound facades).
- Registration/validation stays dev-build-gated where relevant; this slice adds no new dev-only validation (the `when` policy is pure runtime filtering, not a construction-time check).
- JSDoc conventions: see `[[feedback-jsdoc-conventions]]` memory — every public export gets a doc comment; `@internal` for engine-only surface.
- `pnpm docs:api:generate` must be run and committed before the final push (push-gated).
- Every task ends green on its own scoped test command before moving to the next.
- **Documented scope boundary (not an oversight):** the transition gate (`_inputGateDepth`) is tied to the _explicit fade-transition_ lifecycle (`_transition !== null`), matching the definition spec's literal wording ("while a transition runs" / "suppresses all scene input policies... during the transition"). A `setScene()` call **without** a `transition` option gets no equivalent protection against the narrow window where the incoming scope's root is already attached (during `prepare()`) but its scope has not yet flipped to `Active` — closing that generally would require tracking per-root scope ownership inside `InteractionManager`, which is out of scope here. Document this plainly in code; do not silently paper over it.

## Task 1: `SceneDirector` — `_inputGateDepth` transition gate

**Files:**

- Modify: `src/core/SceneDirector.ts`
- Test: `test/core/scene-director.test.ts`

**Interfaces:**

- Produces: `SceneDirector._inputGateDepth: number` (private field); `/** @internal */ get _transitionGateOpen(): boolean` (public getter, `@internal`-documented) returning `this._inputGateDepth > 0`. Consumed by Task 3 (`SceneInputs`) and Task 4 (`InteractionManager`).

The counter opens exactly when a fade transition begins (`this._transition` is assigned inside the `Promise` executor in `_runWithTransition`) and closes on every one of the three places `this._transition` is set back to `null`: `_finishTransition()` (success), `_executeTransitionAction()`'s catch block (mid-transition failure), `destroy()`'s early-reject block (director destroyed mid-transition). A depth counter (not a boolean) matches the spec's exact naming (`_inputGateDepth`) even though only one transition can run at a time today (`_runWithTransition` throws "Scene transition is already in progress" otherwise) — this keeps the door open for a future second gate source without a redesign, and increment/decrement is symmetric and easy to audit at exactly 1 open site + 3 close sites.

## Task 3: `SceneInputs` — `when` policy + edge rules

**Files:**

- Modify: `src/core/scene/SceneInputs.ts`
- Modify: `test/core/scene/scene-inputs.test.ts`

**Interfaces:**

- Consumes: `this._getState()` (Task 2), `this._app.scenes._transitionGateOpen` (Task 1).
- Produces: `SceneInputAvailability = 'active' | 'paused' | 'always'`; `SceneInputBindingOptions extends InputBindingOptions { when?: SceneInputAvailability }`; all four factories (`onStart`/`onActive`/`onStop`/`onTrigger`) accept `SceneInputBindingOptions` instead of `InputBindingOptions`. Exported from `src/core/index.ts` (type only).

Design (per definition §13.1–13.2, implementation §8.1): `whenPolicyAllows(when, state)` returns `false` for `Preparing`/`Suspended`/`Destroying`/`Destroyed` regardless of `when`; for `Active`/`Paused` it checks `when` (`'active'` ⇒ Active only, `'paused'` ⇒ Paused only, `'always'` ⇒ either). Combined with `!this._app.scenes._transitionGateOpen` and a per-facade `_suspended` flag (wired now so Slice E's retention doesn't have to touch this file again — `suspend()`/`resume()` already exist as stubs; this task makes them real for the `when`-gating path specifically, while leaving the _un-bind-everything_ semantics unchanged).

Edge rule (§13.2): a binding fires `onTrigger` only when both the press (`onStart`) and release (`onStop`/`onTrigger`) edges occurred in an allowed state. Implementation: track a per-binding `primed` flag. `onStart`: `primed = allowed now`; invoke the user callback only if `primed`. `onActive`: re-check `allowed now` on every tick (it fires every frame while held) — if now disallowed, set `primed = false` (this is the "reset the moment availability flips allowed → disallowed" rule) and suppress; otherwise invoke only if still `primed`. `onStop`/`onTrigger`: invoke only if `primed && allowed now` (this is what actually enforces "both edges must be allowed" — the release edge is checked live here, not just inferred from the stale `primed` flag, which alone would miss a same-frame pause-then-release with no intervening `onActive` tick).

```ts
import type { Application } from '#core/Application';
import { SceneState } from '#core/SceneState';
import type { Destroyable } from '#core/types';
import type { InputBinding, InputBindingOptions, InputChannel } from '#input/InputBinding';

/** Visible-scene states in which a {@link SceneInputs} binding may dispatch. Default: `'active'`. */
export type SceneInputAvailability = 'active' | 'paused' | 'always';

/** Construction options for every {@link SceneInputs} factory method. */
export interface SceneInputBindingOptions extends InputBindingOptions {
  /** Visible scene states in which this binding may dispatch. Default: `'active'`. */
  readonly when?: SceneInputAvailability;
}

const gatedStates = new Set<SceneState>([SceneState.Preparing, SceneState.Suspended, SceneState.Destroying, SceneState.Destroyed]);

function whenPolicyAllows(when: SceneInputAvailability, state: SceneState): boolean {
  if (gatedStates.has(state)) {
    return false;
  }

  if (when === 'always') {
    return true;
  }

  return when === 'active' ? state === SceneState.Active : state === SceneState.Paused;
}

/**
 * Scene-bound input facade. Bindings created here are automatically unbound
 * when the owning scene ends permanently. Access via {@link Scene.inputs}.
 *
 * Delegates to `app.input` for every binding — this facade adds no second
 * input clock, it only tracks what it created so it can unbind on teardown.
 * Every factory accepts a `when` option (default `'active'`) controlling
 * which {@link SceneState}s the binding may dispatch in; a trigger only
 * fires when both its press and release edges occurred in an allowed state
 * (definition §13.2) — see the class-level suite in
 * `test/core/scene/scene-inputs.test.ts` for the full matrix.
 */
export class SceneInputs implements Destroyable {
  private readonly _bindings = new Set<InputBinding>();
  private _suspended = false;

  public constructor(
    private readonly _app: Application,
    private readonly _getState: () => SceneState,
  ) {}

  public onStart(channel: InputChannel | readonly InputChannel[], callback: (value: number) => void, options?: SceneInputBindingOptions): InputBinding {
    return this._bind('onStart', channel, callback, options);
  }

  public onActive(channel: InputChannel | readonly InputChannel[], callback: (value: number) => void, options?: SceneInputBindingOptions): InputBinding {
    return this._bind('onActive', channel, callback, options);
  }

  public onStop(channel: InputChannel | readonly InputChannel[], callback: (value: number) => void, options?: SceneInputBindingOptions): InputBinding {
    return this._bind('onStop', channel, callback, options);
  }

  public onTrigger(channel: InputChannel | readonly InputChannel[], callback: (value: number) => void, options?: SceneInputBindingOptions): InputBinding {
    return this._bind('onTrigger', channel, callback, options);
  }

  /**
   * Disable every tracked binding's dispatch without unbinding it. Reserved
   * for retention (suspend/resume). Independent of the `when` policy — a
   * suspended facade dispatches nothing regardless of `when`.
   */
  public suspend(): void {
    this._suspended = true;
  }

  /** Restore normal `when`-policy dispatch after {@link SceneInputs.suspend}. */
  public resume(): void {
    this._suspended = false;
  }

  public destroy(): void {
    for (const binding of this._bindings) {
      binding.unbind();
    }

    this._bindings.clear();
  }

  private _bind(
    kind: 'onStart' | 'onActive' | 'onStop' | 'onTrigger',
    channel: InputChannel | readonly InputChannel[],
    callback: (value: number) => void,
    options?: SceneInputBindingOptions,
  ): InputBinding {
    const when = options?.when ?? 'active';
    const forwarded: InputBindingOptions | undefined = options === undefined ? undefined : { threshold: options.threshold, gamepadSlot: options.gamepadSlot };

    let primed = false;

    const allowedNow = (): boolean => !this._suspended && !this._app.scenes._transitionGateOpen && whenPolicyAllows(when, this._getState());

    const wrapped = (value: number): void => {
      switch (kind) {
        case 'onStart': {
          primed = allowedNow();

          if (primed) {
            callback(value);
          }

          return;
        }
        case 'onActive': {
          if (!allowedNow()) {
            primed = false;

            return;
          }

          if (primed) {
            callback(value);
          }

          return;
        }
        case 'onStop':
        case 'onTrigger': {
          if (primed && allowedNow()) {
            callback(value);
          }

          if (kind === 'onStop') {
            primed = false;
          }

          return;
        }
      }
    };

    const binding = this._app.input[kind](channel, wrapped, forwarded);

    this._bindings.add(binding);

    return binding;
  }
}
```

```ts
import type { Application } from '#core/Application';
import type { SceneState } from '#core/SceneState';
import type { Destroyable } from '#core/types';
import type { RenderNode } from '#rendering/RenderNode';

/**
 * Handle returned by {@link SceneInteraction.observe}. Detaches the observed
 * root from interaction dispatch — call {@link InteractionObservation.release}
 * (or {@link InteractionObservation.destroy}, an alias) when the root no
 * longer needs pointer/focus routing. Idempotent; also released automatically
 * when the owning scene ends permanently.
 */
export interface InteractionObservation extends Destroyable {
  /** Detach the observed root. Idempotent alias for {@link InteractionObservation.destroy}. */
  release(): void;
}

/**
 * Handle returned by {@link SceneInteraction.capture}. While active, pointer
 * hit-testing is confined to the captured root's subtree. Call
 * {@link InteractionCapture.release} (or {@link InteractionCapture.destroy},
 * an alias) to end the capture — nested captures restore whichever capture
 * was active before this one, regardless of release order. Idempotent; also
 * released automatically when the owning scene ends permanently.
 */
export interface InteractionCapture extends Destroyable {
  /** `true` until this capture is released. */
  readonly active: boolean;
  /** End this capture. Idempotent alias for {@link InteractionCapture.destroy}. */
  release(): void;
}

interface TrackedObservation extends InteractionObservation {
  readonly root: RenderNode;
  released: boolean;
}

interface TrackedCapture extends InteractionCapture {
  readonly root: RenderNode;
  released: boolean;
}

/**
 * Scene-bound interaction facade. `scene.root` and a materialized `scene.ui`
 * are attached automatically at activation and detached at teardown — that
 * automatic wiring lives in the internal `SceneScope`, not here.
 * {@link SceneInteraction.observe} is the *explicit* path for additional
 * roots (e.g. a subtree rendered outside `scene.root`); {@link
 * SceneInteraction.capture} confines hit-testing to one subtree (modal
 * dialogs, pause menus). Access via {@link Scene.interaction}.
 *
 * Delegates entirely to `app.interaction` — no second picking/dispatch
 * engine, just tracking of what this facade attached/pushed so it can
 * detach/release on teardown.
 */
export class SceneInteraction implements Destroyable {
  private readonly _observations = new Set<TrackedObservation>();
  private readonly _captures: TrackedCapture[] = [];
  private _suspended = false;

  public constructor(
    private readonly _app: Application,
    private readonly _getState: () => SceneState,
  ) {}

  /**
   * Attach `root` to interaction dispatch (pointer/focus routing), so its
   * interactive descendants start receiving events. Returns a handle to
   * detach it early; otherwise it is detached automatically when the scene
   * ends permanently.
   */
  public observe(root: RenderNode): InteractionObservation {
    this._app.interaction.attachRoot(root);

    const observation: TrackedObservation = {
      root,
      released: false,
      release: () => this._release(observation),
      destroy: () => this._release(observation),
    };

    this._observations.add(observation);

    return observation;
  }

  /**
   * Confine pointer hit-testing to `root`'s subtree until the returned
   * handle is released — a modal dialog, pause menu, or full-screen overlay
   * that must swallow clicks outside itself. Nested captures use
   * last-created priority; releasing any capture (not only the most recent)
   * restores the stack to its state as if that capture had never been
   * created, preserving the relative order of the rest.
   */
  public capture(root: RenderNode): InteractionCapture {
    this._app.interaction.pushInputCapture(root);

    const capture: TrackedCapture = {
      root,
      released: false,
      release: () => this._releaseCapture(capture),
      destroy: () => this._releaseCapture(capture),
      get active(): boolean {
        return !this.released;
      },
    };

    this._captures.push(capture);

    return capture;
  }

  /**
   * Disable every tracked observation and capture without detaching/
   * releasing it. Reserved for retention (suspend/resume) — a later slice
   * wires this to actual suspend/restore transitions.
   * @internal
   */
  public suspend(): void {
    this._suspended = true;
  }

  /** Restore normal dispatch after {@link SceneInteraction.suspend}. @internal */
  public resume(): void {
    this._suspended = false;
  }

  public destroy(): void {
    for (const observation of [...this._observations]) {
      this._release(observation);
    }

    this._observations.clear();

    for (const capture of [...this._captures].reverse()) {
      this._releaseCapture(capture);
    }
  }

  private _release(observation: TrackedObservation): void {
    if (observation.released) {
      return;
    }

    observation.released = true;
    this._observations.delete(observation);
    this._app.interaction.detachRoot(observation.root);
  }

  private _releaseCapture(capture: TrackedCapture): void {
    if (capture.released) {
      return;
    }

    const index = this._captures.indexOf(capture);

    // Pop every still-active capture from the top down through (and
    // including) this one, then re-push everything that was above it, in
    // original order — restores the manager's stack as if `capture` had
    // never existed, without needing a manager-level "remove at index" API.
    const above = this._captures.slice(index + 1);

    for (let i = this._captures.length - 1; i >= index; i--) {
      this._app.interaction.popInputCapture();
    }

    capture.released = true;
    this._captures.splice(index, 1);

    for (const entry of above) {
      this._app.interaction.pushInputCapture(entry.root);
    }
  }
}
```
