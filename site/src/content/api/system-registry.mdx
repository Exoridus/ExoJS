## Import

`import { SystemRegistry } from '@codexo/exojs'`

Ordered registry of per-frame Systems, shared by Scene (as
`scene.systems`) and `Application` (as `app.systems`). Systems tick once per
frame in ascending `order` (ties keep insertion order) and are destroyed in
reverse registration order when the registry is destroyed.

Structural mutations made from inside a system's `update()` (a system adding
or removing another, or itself) are deferred until the tick completes, so the
iteration is never invalidated mid-frame.

## Constructors

- `new(): SystemRegistry`

## Methods

- `add(system: T): T` - Add system; returns it for fluent capture. Ticks from the next frame.
- `destroy(): void`
- `has(system: System): boolean` - Whether system is registered.
- `remove(system: System): boolean` - Remove system without destroying it. Returns true if it was registered.

## Properties

- `size: number` - Number of registered systems.

## Events

- `onAdd: Signal<[system]>` - Dispatched when a system is added to this registry.
- `onRemove: Signal<[system]>` - Dispatched when a system is removed from this registry.
