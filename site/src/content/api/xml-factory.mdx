## Import

`import { XmlFactory } from '@codexo/exojs'`

AssetFactory implementation that loads XML files and produces a
parsed Document via the browser's built-in DOMParser.

Throws if the response body is not well-formed XML. Use the returned
`Document` directly with standard DOM APIs — `doc.querySelector`,
`doc.getElementsByTagName`, etc. — or pass it to a format-specific parser
(e.g. for Tiled TMX, BMFont, or TexturePacker XML atlas files).

## Constructors

- `new(): XmlFactory`

## Methods

- `create(source: string): Promise<Document>` - Parses the XML string into a Document. Throws if the browser's DOMParser reports a parse error (indicated by a \<parsererror> element in the returned document).
- `createObjectUrl(blob: Blob): string` - Creates an object URL from blob and registers it for automatic cleanup when destroy is called. Prefer this over calling URL.createObjectURL directly inside a factory subclass so the URL is always tracked.
- `destroy(): void` - Revokes every object URL that was created via createObjectUrl and clears the internal tracking pool.
- `process(response: Response): Promise<string>` - Reads the response body as a UTF-8 string containing the raw XML markup.
- `revokeObjectUrl(objectUrl: string): void`

## Properties

- `storageName: "xml"` - Identifier used as the object-store / storage-namespace key when this factory's assets are persisted to a CacheStore.
