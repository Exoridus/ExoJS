## Import

`import { BeatDetector } from '@codexo/exojs-audio-fx'`

Real-time tempo + beat tracker. Splits work between the audio-rendering
thread (an AudioWorklet that runs onset detection, tempogram analysis,
and parallel 3/4 and 4/4 posterior estimation) and the main thread (this
class ŌĆö receives beats, fires Signals, handles configuration and source
routing).

Accepts a wide range of BeatDetectorSources ŌĆö a bus, an individual
Voice, a raw MediaStream, or any AudioNode ŌĆö and exposes a Signal
for each notable event:
- BeatDetector.onBeat ŌĆö every detected beat
- BeatDetector.onDownbeat ŌĆö first beat of each bar
- BeatDetector.onBarStart ŌĆö bar boundary
- BeatDetector.onTempoChange ŌĆö when the tracked BPM changes
- BeatDetector.onBeatPredicted ŌĆö look-ahead schedule notice

Early beats arrive with low latency tagged `status: 'provisional'` (after a
short `minSettlingMs` warm-up, default 400 ms) for snappy visual reactivity,
then promote to `status: 'locked'` once the tempo grid is trustworthy ŌĆö a
single BeatDetector.onBeat signal carries both via BeatInfo.status.
Set `emitProvisionalBeats: false` to receive only locked beats. Time-signature
detection (3/4 vs 4/4) is on by default; lock to 4/4 by setting
`enableTimeSignatureDetection: false`.
