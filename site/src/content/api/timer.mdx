## Import

`import { Timer } from '@codexo/exojs'`

Clock variant with a fixed limit. Inherits start/stop/reset/restart
semantics; adds Timer.expired (true once `elapsedTime >= limit`)
and remaining-time accessors. Useful for cooldowns, delays, and any timed
gating logic where you want to ask "is the duration up?" each frame.

## Constructors

- `new(limit: Time, autoStart: boolean): Timer`

## Methods

- `destroy(): void`
- `reset(): this` - Halt and zero the accumulated time. The clock is left stopped.
- `restart(): this` - Reset accumulated time to zero, then immediately start. Common per-frame pattern.
- `start(): this` - Begin accumulating time. No-op when already running.
- `stop(): this` - Halt accumulation. Elapsed time stays at the moment of stopping.

## Properties

- `elapsedHours: number`
- `elapsedMilliseconds: number`
- `elapsedMinutes: number`
- `elapsedSeconds: number`
- `elapsedTime: Time` - Total accumulated time since the last Clock.reset. While the clock is running, the value advances on every read by folding in the delta since the previous read; while stopped, the value is fixed at the stop point. Returns the same Time instance — read the scalar fields if you need an unchanging snapshot.
- `expired: boolean` - true once the elapsed time has reached or exceeded the configured limit.
- `limit: Time`
- `remainingHours: number`
- `remainingMilliseconds: number`
- `remainingMinutes: number`
- `remainingSeconds: number`
- `running: boolean`
