## Import

`import { SoundFactory } from '@codexo/exojs'`

AssetFactory implementation that loads short audio assets
(MP3, OGG, WAV, AAC, and other Web Audio API-supported formats), fully
decodes them into an AudioBuffer, and produces a Sound
instance ready for low-latency playback.

For long-form background music use MusicFactory instead, which
streams audio without up-front decoding. Supports audio sprite sheets via
the `sprites` option.

- `new(): SoundFactory`

## Methods

- `create(source: ArrayBuffer, options: SoundFactoryOptions): Promise<Sound>` - Fully decodes the audio buffer via the shared AudioContext and constructs a Sound with the given options.
- `createObjectUrl(blob: Blob): string` - Creates an object URL from blob and registers it for automatic cleanup when destroy is called. Prefer this over calling URL.createObjectURL directly inside a factory subclass so the URL is always tracked.
- `destroy(): void` - Revokes every object URL that was created via createObjectUrl and clears the internal tracking pool.
- `process(response: Response): Promise<ArrayBuffer>` - Reads the full response body as an ArrayBuffer for decoding by the Web Audio API.
- `revokeObjectUrl(objectUrl: string): void`

## Properties

- `storageName: "sound"` - Identifier used as the object-store / storage-namespace key when this factory's assets are persisted to a CacheStore.
