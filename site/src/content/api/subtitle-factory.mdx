## Import

`import { SubtitleFactory } from '@codexo/exojs'`

AssetFactory implementation that parses WebVTT (`.vtt`) and SubRip
(`.srt`) subtitle files and produces an array of VTTCue instances.

Format is detected from the file extension of the response URL; unknown
extensions default to VTT parsing. VTT cue settings (`align`, `line`,
`position`, `size`, `vertical`) are applied; SRT cues retain default
positional properties.

## Methods

- `create(source: SubtitleIntermediate): Promise<VTTCue[]>` - Parses the subtitle text into an ordered array of VTTCue instances.
- `createObjectUrl(blob: Blob): string` - Creates an object URL from blob and registers it for automatic cleanup when destroy is called. Prefer this over calling URL.createObjectURL directly inside a factory subclass so the URL is always tracked.
- `destroy(): void` - Revokes every object URL that was created via createObjectUrl and clears the internal tracking pool.
- `process(response: SubtitleSource): Promise<SubtitleIntermediate>` - Reads the response body as UTF-8 text and records the subtitle format derived from the response URL's file extension.
- `revokeObjectUrl(objectUrl: string): void`

## Properties

- `storageName: "subtitle"` - Identifier used as the object-store / storage-namespace key when this factory's assets are persisted to a CacheStore.
