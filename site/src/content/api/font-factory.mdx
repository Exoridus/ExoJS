## Import

`import { FontFactory } from '@codexo/exojs'`

AssetFactory implementation that loads web-font binary data (WOFF,
WOFF2, TTF, OTF) and produces a parsed FontFace.

By default each created font is registered with `document.fonts`, making it
globally available to CSS and Canvas without any further steps. Fonts added
this way are automatically removed from `document.fonts` when
FontFactory.destroy is called.

## Constructors

- `new(): FontFactory`

## Methods

- `create(source: ArrayBuffer, options?: FontFactoryOptions): Promise<FontFace>` - Constructs and loads a FontFace from binary font data. Throws if options.family is not supplied, or if the buffer contains fewer than 4 bytes or is otherwise unrecognised as a valid font. Unless options.addToDocument is explicitly false, the font face is registered via document.fonts.add() immediately after loading.
- `createObjectUrl(blob: Blob): string` - Creates an object URL from blob and registers it for automatic cleanup when destroy is called. Prefer this over calling URL.createObjectURL directly inside a factory subclass so the URL is always tracked.
- `destroy(): void` - Removes all font faces previously registered with document.fonts by this factory, then releases any tracked object URLs via the base AbstractAssetFactory.destroy implementation.
- `process(response: Response): Promise<ArrayBuffer>` - Reads the full response body as an ArrayBuffer suitable for passing to the FontFace constructor.
- `revokeObjectUrl(objectUrl: string): void`

## Properties

- `storageName: "font"` - Identifier used as the object-store / storage-namespace key when this factory's assets are persisted to a CacheStore.
