## Import

`import { CsvFactory } from '@codexo/exojs'`

AssetFactory implementation that loads CSV (and TSV) files and
produces a two-dimensional array of strings.

Parsing is RFC 4180-compliant: quoted fields (including fields that contain
the delimiter, newlines, or double-quotes) are handled correctly. Line
endings are normalised before parsing; empty trailing lines are dropped.

The returned `string[][]` is intentionally untyped — callers decide whether
to treat the first row as headers, apply type coercions, etc.

## Methods

- `create(source: string, options: CsvFactoryOptions): Promise<string[][]>` - Parses the CSV text into a two-dimensional array of strings. Each element of the outer array is one row; each element of an inner array is one field value with surrounding quotes stripped and escaped double-quotes ("") unescaped.
- `createObjectUrl(blob: Blob): string` - Creates an object URL from blob and registers it for automatic cleanup when destroy is called. Prefer this over calling URL.createObjectURL directly inside a factory subclass so the URL is always tracked.
- `destroy(): void` - Revokes every object URL that was created via createObjectUrl and clears the internal tracking pool.
- `process(response: Response): Promise<string>` - Reads the response body as a UTF-8 string.
- `revokeObjectUrl(objectUrl: string): void`

## Properties

- `storageName: "csv"` - Identifier used as the object-store / storage-namespace key when this factory's assets are persisted to a CacheStore.
