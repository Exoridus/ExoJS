## Import

`import { PassContext } from '@codexo/exojs'`

Pass-scoped DrawContext: the active render target and view for a
CallbackRenderPass callback. `clear`/`render`/`renderTo` route through
the owning RenderingContext (and its pass coordinator), so they act on
THIS pass's target and never reset the active view or leak onto another
target. Default view for `render`/`draw*` is the pass view â€” not the world
view â€” so a callback inside an off-screen redirect renders correctly without
passing `\{ view \}` explicitly.

Use backend only for raw draws or GPU state; switching target / view /
clear through it bypasses the coordinator (the leak this type prevents).

## Constructors

- `new(_context: RenderingContext, target: RenderTarget, view: View): PassContext`

## Methods

- `clear(color: Color): void` - Clear the active target to color (coordinator-routed; no leak).
- `drawBatch(batch: RenderBatch, options?: DrawBatchOptions): void` - Immediately draw an instanced batch.
- `drawGeometry(geometry: Geometry, transform: Matrix, options?: DrawGeometryOptions): void` - Immediately draw a single geometry with transform as its world matrix.
- `render(node: RenderNode, options?: RenderOptions): void` - Render node into the active target with options.view (default: the active view).
- `renderTo(node: RenderNode, options: RenderToOptions): void` - Render node into a caller-owned off-screen target (nested when called inside a pass).

## Properties

- `target: RenderTarget` - The target this pass draws into (read-only).
- `view: View` - The view this pass renders with (read-only).
- `backend: RenderBackend` - Raw backend â€” draws/state only; do NOT switch target/view/clear here.
